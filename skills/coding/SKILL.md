# Coding Best Practices

Use this skill when implementing or reviewing code.

## Guidelines

- Inspect before editing.
- Prefer small reversible changes.
- Keep public APIs simple.
- Add tests for behavior changes.
- Avoid unnecessary dependencies.
- Keep secrets out of code and logs.
- Leave a short summary of what changed and why.

## Workflow

1. Understand the task.
2. Locate relevant files.
3. Read existing conventions.
4. Make the smallest useful change.
5. Run tests or checks when available.
6. Report the final diff and next steps.
