# UI/UX Review

Use this skill when reviewing screens, components, flows, or visual systems.

## Checklist

- Is the primary action obvious?
- Are empty, loading, and error states handled?
- Is keyboard navigation supported?
- Are colors high contrast?
- Are interactive targets large enough?
- Does the UI avoid unnecessary motion?
- Does the layout work on smaller screens?
- Are states named clearly?

## Output

Return:

1. What works well
2. Issues by severity
3. Concrete fixes
4. Optional polish suggestions
